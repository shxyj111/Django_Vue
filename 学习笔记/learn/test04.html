三、属性选择器（Attribute Selectors）

[attr] 存在 [=] 等于 [~=] 列表含 [^=] 开头 [$=] 结尾 [*=] 包含 [|=] 前缀

① [data-flag] 存在属性即命中：我有 data-flag（紫框） 我无（不变）
② [data-role="admin"]：admin（红底） user（不变）
③ [data-tags~="vue"]（空格列表含 vue）：vue js（绿） react（不变）
④ a[href^="https"] / a[href$=".pdf"]： https 链接 🔒 · doc.pdf (PDF) · page.html（无后缀）
⑤ [data-note*="重要"] 包含：含"重要"（红左边框） 普通（不变）
⑥ [lang|="zh"] 前缀：zh-CN（斜体） en（不变）

七、Vue scoped 原理（概念示意）

这不是浏览器原生选择器，而是 Vue 编译期的"语法糖"。下面用普通 HTML 模拟它编译后的结果。

[data-v-demo] .scoped-p（模拟 scoped：仅当前组件内的 .scoped-p 变紫）：

我在 data-v-demo 容器内（紫）

我在容器外（不变，证明 scoped 隔离）

真实 Vue 组件中写 <style scoped>，编译后会变成类似 .scoped-p[data-v-7a9e1f] { ... }，并给该组件模板每个元素自动加上 data-v-7a9e1f 属性，从而实现"样式只作用于本组件、不污染全局"。 所以 class 只是选择器之一，scoped 的本质是"属性选择器 + 编译期自动加属性"。
